1) BROWSER - When a user enters a URL, the browser sends a full HTTP request, not just the URL.
    eg HTTP request is like:-
    GET /shorten?url=https://google.com HTTP/1.1
    Host: localhost:8080
    User-Agent: Chrome
    Accept: */*
    this is sent to tomcat

2)TOMCAT -is simply the HTTP server.
    Its job:
    Receive HTTP request
    Parse it
Forward it to Spring

3)DispatchServlet - Router
    User requests:
    GET /shorten
    DispatcherServlet searches for:
    @GetMapping("/shorten")
    in your controllers.

4) CONTROLLER - contains api for different functions
    Controller translates HTTP в†’ Java function call.
    POST /shorten
    url=https://google.com
    Controller convert this HTTP request to Java code
    urlService.shorten("https://google.com")
